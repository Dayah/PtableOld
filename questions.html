General Questions

Who made this periodic table?
Michael Dayah.
Why?
He thought it would look nice converted to HTML and wanted to see if he could do it. He was also bored while staying at home recovering from toe surgery.
When?
September 1997. It was posted on the Internet October 1, 1997. Element descriptions were added later in December 1997.
What's the big deal? There are other periodic tables on the Internet.
Not only is this one of the very few tables that has all the information on one page, it's quite possibly the most complex use of HTML tables anywhere on the Internet. This periodic table was made with approximately eight hundred lines of HTML, no images.
How long did it take to make?
A total of about five hours plus one half hour per month maintenance.
From where was the look of the table taken?
The color scheme is loosely based on a Frank Schaffer Publications poster. The categories were also used from this poster. The idea of using white space between elements was taken from the periodic table in the designer's biology book.
Can I print it?
Yes. You may wish to resize or resample the image to fit on a page using your graphics program.
Is there a license agreement I must follow?
Because of the frequent piracy of the periodic table, you are not allowed to redistribute the table electronically in any fashion. You, however, are allowed to print the table out for your own, personal use. If you would like to print out many copies for a classroom, please tell me.
You may also obtain a high-gloss print or transparency of the periodic table. Simply send five US dollars to Michael Dayah, 2420 Congress Parkway #107, Athens, TN 37303, with your e-mail address.
Can I link to your periodic table?
By all means. You can distinguish it from other periodic tables by linking it as "Michael Dayah's Periodic Table" or "LuceNT's Periodic Table".

HTML-Related Questions

The table looks distorted or is indecipherable.
Consider upgrading your browser. Below is a description of which browsers can display which aspects of the page. The HTML 4 compliant version of the page looks a bit messy even in the 5.0 browsers because they still do not fully support CSS. If the paragraph tells you your browser is "sufficient", and it still looks distorted, please e-mail me with your browser and version.
Which browsers can display the table?
All structure, font style, and size are shown completely in the 4.0 versions of Microsoft Internet Explorer and Netscape Navigator. Structure, but not font style or size is preserved in Navigator 3.0. Because of the way ROWSPAN and COLSPAN overlap to save markup, Internet Explorer 3.0 (and 4.0 for the Mac) do not display the periodic table correctly. Navigator 2.0 displays the table correctly, minus the cell background colors. Internet Explorer 2.0 does not support HTML tables. The table appears vastly distorted in Opera 3.5 but quite well in 3.6. To an extent, the information can even be extracted with Lynx.
What editor was used to make the periodic table?
Only notepad. No WYSIWYG or enhanced text editors were used.
It could have been done in FrontPage, HotDog, or [insert WYSIWYG HTML editor here].
Probably not. But if it could, it would not be as cross-browser compatible as it is now and would take up at least twice the markup.
What do you mean by "complex tables"?
There are four qualities which make a table complex:
  • The table uses many dependent columns and rows.
  • The structure and markup cannot be optimized any further.
  • COLSPAN and ROWSPAN are used extensively.
  • Removing one cell will likely cause the table to collapse.
But table XYZ is more complex.
If you find a table more complex, please tell me about it.
What does it look like with table borders turned on?
See for yourself.
How many <TABLE>'s were used in the creation of the table?
Structurally, only two tables were needed. For cosmetic reasons, there are sixteen tables total.
How deep are the tables nested?
Structurally, only one level of nesting was required. A second level was used for purely cosmetic reasons.
Are there any images?
No.
The HTML looks proprietary.
One proprietary element was used to make the table look identical in both Navigator and Internet Explorer. This is the same reason the table does not validate using W3C's validator. The designer prioritizes usability over validation.
If you would like to see what the periodic table looks like when no design "tricks" are used to make the table browser-compatible, look at the optimized version. The optimized table is 12% smaller in file size than the browser-compatible version and is HTML 4 Strict DTD compliant. Though when it was made, no browser could render it correctly. The upcoming 5.0 browsers should render it identically to the browser-compatible table.
Of which web technologies does the table take advantage?
HTML 4.0, Cascading Style Sheets (level one and two), and JavaScript.
